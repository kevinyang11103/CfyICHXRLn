# 基于Web的旅行推荐系统简介

## 核心功能
- 用户注册与登录
  - 多途径注册：支持邮箱、手机号注册，设置密码复杂度校验规则，确保密码安全。
  - 密码管理：提供密码找回、修改功能，通过邮箱或手机验证码验证身份。
- 搜索与发现功能
  - 综合搜索：在搜索框输入关键词，可快速查找景点、美食、路线等信息。搜索结果按相关性、热度排序展示。
  - 二级联动筛选：发现板块设置省份和城市二级联动下拉框，选择城市后，页面自动展示该地区热门景点、美食图片轮播、旅游景点地铁路线图，可点击放大查看。同时推荐小众旅游地和美食，点击可进入对应分类。
  - 景点详情展示：点击景点进入系统百科页面，侧边栏定位文章内容，右侧推荐其他景点，方便用户跳转浏览。
- 路线规划功能
  - 自主规划：寻迹板块支持用户输入出发地和目的地，点击生成路线，借助百度API实现地图展示，实时生成导航路线。
  - 热门推荐：页面下方展示热门路线推荐板块，为用户推荐可能感兴趣的路线地图。侧边栏可收缩，鼠标悬停弹出，方便切换页面。
- 社交互动功能
  - 论坛互动：用户在论坛界面可发布旅游相关帖子，其他用户能对帖子进行点赞、收藏、评论。设置轮播展示图，展示热门或优质帖子。
  - 信息反馈：搭建评论、评分体系，用户对景点、路线等进行评价，为其他用户提供参考。
- 数据管理功能
  - 用户数据管理：在 user 表存储用户信息，保障数据安全，定期备份。
  - 景点数据管理：province 表存储省份、城市、景点信息；cold 表和 hot 表分别存储小众和热门景点信息。管理员可在后台对这些数据进行增删改查操作，及时更新景点信息。
  - 其他数据管理：like 和 collect 表记录点赞和收藏数据，map 表保存地图信息，支持地图信息的更新与维护。

## 适用场景
- 旅行爱好者
- 旅游行业从业者
- 地图API开发者
- 数据分析师

## 技术特点
- 基于Web的旅行推荐系统，支持多平台访问。
- 结合百度地图API实现实时路线规划。
- 采用数据管理功能，确保数据安全、及时更新。
- 社交互动功能，促进用户交流与分享。
